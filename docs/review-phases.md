# 리뷰 단계

## 1. 입력 및 파싱
- 문자열 리터럴 안의 세미콜론이 배치를 잘못 나누지 않는지 확인한다.
- 마지막 세미콜론 규칙과 빈 문장 거부가 정확한지 확인한다.
- lexer 토큰 범위와 키워드 대소문자 무시 처리가 맞는지 확인한다.
- parser 오류가 실제 실패 지점을 가리키는지 확인한다.

## 2. 저장소와 캐시
- `books.bin`이 없을 때 첫 로딩 동작이 올바른지 확인한다.
- 바이너리 헤더 검증과 고정 길이 행 읽기가 정확한지 확인한다.
- 임시 파일 저장 후 교체 흐름이 안전한지 확인한다.
- 메모리 캐시 값과 실제 저장된 파일 값이 일치하는지 확인한다.

## 3. 인덱스와 실행기
- `WHERE id = n`이 실제로 B+ 트리를 타는지 확인한다.
- `author`, `genre`, `title`, 전체 조회가 선형 탐색으로만 처리되는지 확인한다.
- projection 컬럼이 하드코딩 출력이 아니라 파싱된 SQL에서 오는지 확인한다.
- pretty print 표 형식, 행 수, scan 모드, 시간 출력이 올바른지 확인한다.

## 4. 트랜잭션과 롤백
- 배치 전체가 성공하기 전까지 출력이 버퍼링되는지 확인한다.
- 뒤쪽 문장이 실패했을 때 앞쪽 `INSERT`가 메모리에 남지 않는지 확인한다.
- 롤백 시 `next_id`가 원복되는지 확인한다.
- 롤백 후 B+ 트리가 다시 구성되는지 확인한다.

## 5. 테스트와 문서
- 단위 테스트와 기능 테스트가 정상 경로와 오류 경로를 모두 다루는지 확인한다.
- README 예시가 실제 CLI 동작과 맞는지 확인한다.
- 바이너리 포맷 문서가 구현과 일치하는지 확인한다.
- 성능 문서에는 실제 측정값만 적혀 있는지 확인한다.
